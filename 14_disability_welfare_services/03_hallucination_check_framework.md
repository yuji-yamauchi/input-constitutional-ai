# ハルシネーションチェック フレームワーク

> NotebookLMが生成した教育マニュアルの検証用
> 作成日：2026-03-04

---

## チェックの全体方針

NotebookLMはソース資料に基づいて生成するが、以下のリスクがある：

1. **ソース外情報の混入**：学習済み知識からの補完（最も多い）
2. **制度情報の時点ズレ**：旧制度の記述が混入する
3. **事業種別の混同**：A型・B型・移行支援の要件を取り違える
4. **数値・条件の誤り**：人員配置基準・報酬単位等の数値が不正確
5. **因果関係の捏造**：ソースにない因果関係や効果を断定する

## チェック項目一覧

### レベル1：事実の正確性（最重要）

| チェック項目 | 確認方法 | 判定 |
|-------------|---------|------|
| 法令名・条文番号が正確か | e-Govで条文を直接確認 | □ |
| 人員配置基準の数値が正しいか | 指定基準省令と突合 | □ |
| 報酬単位数が正確か | 報酬告示と突合 | □ |
| 加算の算定要件が正しいか | 留意事項通知と突合 | □ |
| 制度改正の施行日が正しいか | 厚労省通知の日付と照合 | □ |
| 「義務」「努力義務」「推奨」の区別が正しいか | 法令の文言（しなければならない／するよう努める等）と照合 | □ |

### レベル2：事業種別の正確な区別

| チェック項目 | よくある誤り | 判定 |
|-------------|-------------|------|
| A型の記述に雇用契約前提が反映されているか | B型の記述と混同 | □ |
| B型に労働基準法の適用を前提とした記述がないか | A型の基準を誤適用 | □ |
| 移行支援の利用期間（原則2年）が正しいか | 期間の誤記 | □ |
| 就労アセスメントの対象が正しく記載されているか | 対象範囲の拡大解釈 | □ |
| A型のスコア方式の評価項目が正確か | 項目の追加・欠落 | □ |
| 各事業の基本報酬の算定方式の違いが正しいか | 方式の混同 | □ |

### レベル3：ソース準拠の検証

| チェック項目 | 確認方法 | 判定 |
|-------------|---------|------|
| 【出典】の表記があるか | 全記述を目視確認 | □ |
| 出典として挙げた資料に該当する記述が実在するか | ソース資料と突合 | □ |
| ソースにない「一般的に～と言われている」等の記述がないか | ソースに根拠がない一般論を検出 | □ |
| 「～が効果的である」等の評価的記述にソース根拠があるか | エビデンスの有無を確認 | □ |
| 「要確認事項」として正直にマークされている箇所があるか | 不明点を隠していないかチェック | □ |

### レベル4：実務との整合性

| チェック項目 | 確認方法 | 判定 |
|-------------|---------|------|
| 記載された手順が現場で実行可能か | 実務経験者によるレビュー | □ |
| 自治体ごとのローカルルールへの注記があるか | 「自治体により異なる」等の注記の有無 | □ |
| 様式やテンプレートの内容が現行のものと一致するか | 実際に使用中の様式と照合 | □ |
| 関係機関の名称・役割が現行制度と一致するか | 組織改編等による変更がないか | □ |

### レベル5：構造・一貫性

| チェック項目 | 確認方法 | 判定 |
|-------------|---------|------|
| 章間で矛盾する記述がないか | 同一テーマの記述を横断的に比較 | □ |
| 用語の定義が統一されているか | 用語集と本文の一致を確認 | □ |
| 対象レベル（新人/中堅/管理者）の区分が適切か | 内容の難易度と指定レベルの整合 | □ |
| 参照先URL・資料名が有効か | リンク切れ・資料名の誤りがないか | □ |

## ハルシネーション発生時の対応フロー

```
疑わしい記述を発見
    ↓
ソース資料で該当箇所を検索
    ↓
┌─ ソースに記述あり → 記述の正確性を照合 → 修正 or 承認
│
├─ ソースに記述なし・曖昧 → 「要確認」とマーク → 一次ソース（法令等）で確認
│
└─ 明らかに誤り → 削除 or 正しい情報で置換 → 修正根拠を記録
```

## 特に注意すべきハルシネーションパターン（就労支援分野特有）

1. **A型の最低賃金に関する記述**：都道府県別の最低賃金額を「全国一律○円」と記載するケース
2. **報酬改定年度の混同**：2021年度改定の内容を2024年度改定として記載するケース
3. **就労定着支援事業と就労移行支援の混同**：別事業であるにもかかわらず一体的に記載するケース
4. **サービス管理責任者の配置基準**：利用者数に対する配置割合の数値誤り
5. **個別支援計画のモニタリング頻度**：事業種別ごとの違いを無視した記述
6. **合理的配慮の「義務化」の適用範囲**：民間事業者への義務化（2024年4月〜）と行政機関の義務を混同

## レベル6：フレームワーク整合性チェック（研究理念との矛盾検出）

本マニュアルは実務中心だが、以下の研究フレームワークに**矛盾する記述**が含まれていないかを検出する。
理論そのものを解説する必要はないが、記述が以下の原則に反していれば修正対象となる。

| チェック項目 | 矛盾する記述の例 | 正しい方向性 | 判定 |
|-------------|-----------------|-------------|------|
| 個人帰属の回避（ICF的相互作用モデル） | 「本人の能力不足により就労困難」 | 環境因子・合理的配慮の提供状況を含めた多面的アセスメント | □ |
| 福祉＝サービス業の認識 | 「善意で支援する」「奉仕の精神」 | 法に基づく契約的サービス提供の枠組み | □ |
| 個別支援計画の実質的意義 | 「行政監査のために作成する」 | 利用者との合意形成プロセスとしての位置づけ | □ |
| 本人主権・意思決定支援 | 「支援者が目標を設定する」 | 本人の意思・選好を起点とした目標設定 | □ |
| 記録の透明性 | 「記録は職員間で共有する（利用者非開示）」 | 利用者のアクセス・確認を前提とした記録管理 | □ |
| 機会提供＋環境調整の一体性 | 「訓練メニューを提供すれば責任完了」 | 機会活用に必要な合理的配慮・環境調整を含めた包括的支援 | □ |
| 障害受容の非強制 | 「障害を受容させる」「受容が進んでいない」 | 本人の自己理解の深化を尊重する側面的支援 | □ |

### フレームワーク整合性チェックの根拠

上記チェック項目は以下の法的・制度的根拠に基づく：

- **障害者権利条約（CRPD）** 第12条（法的能力）、第19条（地域生活）
- **障害者総合支援法** 第1条の2（基本理念）：障害者の意思決定の支援に配慮
- **障害者差別解消法** 第8条：合理的配慮の提供義務（2024年4月〜民間事業者にも義務化）
- **ICF（国際生活機能分類）**：環境因子と個人因子の相互作用モデル
- **福祉サービスの契約構造**：利用者と事業者の準委任契約的関係

これらは独自理論ではなく、国際条約・国内法・WHO分類に基づくものであるため、
公的資料との整合性として検証可能である。
