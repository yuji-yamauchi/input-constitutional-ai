# NotebookLM用プロンプト：障害福祉 就労支援 教育マニュアル作成

> 使い方：このプロンプトをNotebookLMのノートブックに貼り付け、ソース資料をアップロードした状態で実行してください。
> 作成日：2026-03-04

---

## プロンプト本文

```
あなたは障害福祉サービス（就労支援分野）の教育マニュアル作成の専門家です。
アップロードされたソース資料のみに基づいて、以下の仕様に沿った「施設・スタッフ向け教育マニュアル」を作成してください。

━━━━━━━━━━━━━━━━━━━
■ マニュアルの基本情報
━━━━━━━━━━━━━━━━━━━

【対象事業】就労継続支援A型／就労継続支援B型／就労移行支援
【対象読者】全スタッフ共通（新人〜管理者まで段階的にカバー）
【トーン】実務中心。現場ですぐ使える具体性を重視。理論は必要最小限の背景説明にとどめる。
【言語】日本語

━━━━━━━━━━━━━━━━━━━
■ マニュアルの章構成（案）
━━━━━━━━━━━━━━━━━━━

以下の構成で作成してください。各章にはソース資料からの根拠を明示すること。

---

### 第1章：就労支援事業の全体像
- 就労系サービスの位置づけ（障害者総合支援法における体系）
- A型・B型・移行支援の違い（目的・対象者・雇用関係の有無・報酬構造）
- 利用者の流れ（相談→アセスメント→利用開始→支援→移行/定着）

### 第2章：法令・制度の基礎知識
- 障害者総合支援法の基本理念と主要条文
- 指定基準（人員配置・設備・運営基準）の概要
- 報酬の仕組み（基本報酬＋加算の構造）
- 障害者差別解消法と合理的配慮（2024年4月〜義務化）

### 第3章：個別支援計画の作成と運用
- アセスメントの基本手順（就労アセスメントを含む）
- 個別支援計画の構成要素と書き方
- モニタリングの頻度と記録方法
- 支援会議の運営方法

### 第4章：日常の支援業務
- 利用者対応の基本姿勢（意思決定支援・エンパワメント）
- 作業指導・職業訓練の進め方
- コミュニケーション支援の実践
- 体調管理・服薬管理への配慮
- 家族・関係機関との連携

### 第5章：事業種別ごとの実務ポイント
- 【A型】雇用契約・労働法規の遵守、経営改善・スコア方式
- 【B型】工賃向上の取り組み、生産活動の運営
- 【移行支援】企業開拓・マッチング、職場実習の実施、定着支援への接続

### 第6章：リスク管理・権利擁護
- 虐待防止と身体拘束適正化（義務化対応）
- ヒヤリハット・事故報告の体制
- 苦情解決の仕組み
- 個人情報保護の実務

### 第7章：組織運営と人材育成
- サービス管理責任者の役割と責務
- OJT・Off-JTの設計
- 自己評価・第三者評価の活用
- 行政監査への対応

### 第8章：外部連携
- 障害者就業・生活支援センター（なかぽつ）との連携
- ハローワーク・JEEDとの協働
- 医療機関・相談支援事業所との情報共有
- 企業との関係構築

### 付録
- 用語集（略語・専門用語の定義一覧）
- 関連法令・通知の一覧と参照先URL
- 様式集（個別支援計画・モニタリング記録・ヒヤリハット報告書等のテンプレート）
- チェックリスト（新人教育の到達度確認用）

━━━━━━━━━━━━━━━━━━━
■ 出力ルール
━━━━━━━━━━━━━━━━━━━

1. **ソース準拠の原則**：アップロードされた資料に記載のある情報のみを使用すること。資料に記載がない内容については「ソース資料に該当する記述なし」と明記すること。
2. **根拠の明示**：各記述について、どのソース資料のどの部分に基づいているかを【出典：○○（資料名）】の形式で示すこと。
3. **事業種別の区別**：A型・B型・移行支援で異なる内容は必ず区別して記載すること。共通事項は共通であることを明記すること。
4. **実務寄りの記述**：「～すべきである」のような抽象論ではなく、「～の場合は○○を行う」のように具体的な行動レベルで記述すること。
5. **段階的な読者対応**：各章の冒頭に【この章の対象レベル】を記載（例：全員必須／中堅以上推奨／管理者向け）。
6. **最新制度への対応**：法改正・報酬改定の時点を明記し、「○年○月時点の情報」と注記すること。
7. **推測の禁止**：ソースにない情報を推測で補完しないこと。不明点は「要確認事項」として明示すること。

━━━━━━━━━━━━━━━━━━━
■ 禁止事項（フレームワーク整合性ルール）
━━━━━━━━━━━━━━━━━━━

以下の記述・表現は、当事業所の支援理念と矛盾するため使用しないこと。
違反する記述を見つけた場合は、代替表現に置き換えること。

| # | 禁止する記述・考え方 | 理由 | 代替表現の方向性 |
|---|---------------------|------|----------------|
| 1 | 「利用者の能力不足が原因で就労できない」等の個人帰属型記述 | 環境因子・制度的障壁の分析なしに個人に原因を帰属させる記述は、構造的問題の不可視化を招く | 「環境調整・合理的配慮の提供状況を含めて要因分析する」等、ICF的な相互作用モデルに基づく記述 |
| 2 | 「支援してあげる」「面倒をみる」等の保護的・恩恵的表現 | 福祉サービスは善意の恩恵ではなく、法に基づくサービス提供義務の履行である | 「支援を提供する」「契約に基づくサービスを実施する」等の対等な関係性に基づく表現 |
| 3 | 「本人のやる気がない」「意欲が低い」等の表面的な動機評価 | 動機の低下には環境要因・心理的要因・制度的障壁が背景にある可能性を無視している | 「参加意欲に影響する要因（体調・環境・過去の経験等）をアセスメントする」等の多面的評価 |
| 4 | 個別支援計画を「書類作成業務」として扱う記述 | 個別支援計画は利用者との合意に基づく支援の根拠文書であり、形式的な行政対応書類ではない | 「利用者との合意形成のプロセスとして位置づける」等の実質的意義を強調する記述 |
| 5 | 「障害の受容」を支援者が促す記述 | 障害受容は本人の主体的なプロセスであり、外部から強制・誘導するものではない | 「本人の自己理解の深化を側面的に支援する」等の本人主体の表現 |
| 6 | 利用者の記録・情報に対する支援者側の一方的管理を前提とする記述 | 記録は支援プロセスの透明性を確保するものであり、本人のアクセス・確認は権利である | 「記録は利用者が確認できる形で管理し、内容への同意を得る」等の透明性を前提とする記述 |
| 7 | 機会提供のみで支援責任が完了するとの記述 | 機会の提供だけでなく、その機会を活用するための環境調整・配慮の提供もサービス内容に含まれる | 「機会提供に加え、活用に必要な合理的配慮・環境調整を一体的に行う」等の記述 |

━━━━━━━━━━━━━━━━━━━
■ ソース資料の出典表記ルール
━━━━━━━━━━━━━━━━━━━

ソース資料には以下の3種類が含まれる。出典表記を使い分けること。

1. **公的資料（法令・通知・ガイドライン等）**
   → 【出典：○○法第○条】【出典：厚労省通知（○年○月○日）】等、正式な法令番号・通知番号で引用
2. **学術文献・調査報告で出典が明確なもの**
   → 【出典：著者名（年）論文名】で引用
3. **独自の検討資料・研究ノートで一次出典が不明確なもの**
   → 【参考：内部検討資料（○○.yaml）※一次出典の確認が必要】と表記し、ハルシネーションチェック対象であることを明示

━━━━━━━━━━━━━━━━━━━
■ 出力形式
━━━━━━━━━━━━━━━━━━━

- Markdown形式で出力
- 各章ごとに分割して出力（1章ずつ指示に応じて生成）
- 章の冒頭に「この章の目的」「対象レベル」「所要学習時間の目安」を記載
```

## 補足：分割生成時のフォローアッププロンプト

NotebookLMでは長文の一括生成が難しい場合があります。その際は以下のように章単位で指示してください。

```
上記のマニュアル構成に従い、第○章を作成してください。
出力ルール（ソース準拠・根拠明示・推測禁止）を厳守してください。
```

---

## 補足：生成後の品質チェック指示プロンプト

マニュアル生成後、同じNotebookLMノートブック内で以下を実行すると自己チェックが可能です。

```
生成した第○章について、以下の観点でチェックしてください：
1. ソース資料に根拠がない記述はないか？（ある場合は該当箇所を列挙）
2. A型・B型・移行支援の記述が混同されていないか？
3. 法令・制度の引用に誤りがないか？
4. 「～すべき」のような抽象記述が残っていないか？
5. 最新の制度改正（2024年度報酬改定等）が正しく反映されているか？
```
