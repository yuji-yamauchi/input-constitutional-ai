# Input Constitutional AI

**Service of Empowerment (SoE) - Redefining Welfare Service Records**

---

## Project Overview

This repository contains the theoretical foundation and research materials for **Input Constitutional AI**, a novel approach to AI ethics in social welfare services.

### Core Innovation

While Anthropic's Constitutional AI focuses on **Output** (constraining AI responses), this project proposes **Input Constitutional AI** - protecting human rights at the **data recording stage** before AI processing begins.

```
Traditional AI Ethics:  Human → Data → [AI Processing] → [Constitutional AI] → Output
                                                              ↑ Output-side constraint

Input Constitutional AI: Human → [Input CAI] → Data → AI Processing → Output
                                     ↑ Input-side rights protection
```

### Research Theme: SoE (Service of Empowerment)

**SoE** stands for both:
- **Service of Empowerment**: Welfare services that enhance capabilities
- **Service or Exploitation**: Critical lens examining power dynamics in support relationships

---

## Theoretical Framework

### Three-Layer Model (Upper Concept)

| Layer | Focus | Example |
|-------|-------|---------|
| **Visible** | Surface issues | Formalization of employment support |
| **Hidden** | Invisible structures | Service or Exploitation dynamics |
| **Philosophical** | Foundational assumptions | Uncritical adoption of rights concepts |

### Four-Pillar Model (Implementation Frame)

1. **Rights Recovery** - Restoring agency to service users
2. **Supporter Ethics** - Ethical framework for support workers
3. **Regional Human Resource** - Community-based talent development
4. **Scientific Assessment** - Evidence-based evaluation methods

**Relationship**: Three-Layer (philosophy) → Four-Pillar (implementation) → SoE System

# Input Constitutional AI（日本語版）

**Service of Empowerment (SoE) - 福祉サービス記録の再定義**

---

## プロジェクト概要

本リポジトリは、社会福祉サービスにおけるAI倫理の新しいアプローチである **Input Constitutional AI** の理論的基盤と研究資料を収録しています。

### 核心的イノベーション

AnthropicのConstitutional AIが**Output**（AI応答の制約）に焦点を当てるのに対し、本プロジェクトは**Input Constitutional AI**を提案します。これはAI処理が始まる**前の**データ記録段階で人権を保護するアプローチです。

```
従来のAI倫理:     人間 → データ → [AI処理] → [Constitutional AI] → 出力
                                                    ↑ Output側の制約

Input Constitutional AI: 人間 → [Input CAI] → データ → AI処理 → 出力
                                    ↑ Input側の権利保護
```

### 研究テーマ: SoE（Service of Empowerment）

**SoE**は二重の意味を持ちます：
- **Service of Empowerment**: ケイパビリティを高める福祉サービス
- **Service or Exploitation**: 支援関係における権力構造を批判的に検討する視点

---

## 理論的枠組み

### Three-Layer Model（上位概念）

| レイヤー | 焦点 | 例 |
|---------|------|-----|
| **Visible（表層）** | 表面的課題 | 就労支援の形骸化 |
| **Hidden（隠れ層）** | 不可視の構造 | Service or Exploitationの力学 |
| **Philosophical（哲学層）** | 基盤的前提 | 人権概念の無批判的受容 |

### Four-Pillar Model（実装フレーム）

1. **権利回復（Rights Recovery）** - 当事者の主体性を取り戻す
2. **支援者倫理（Supporter Ethics）** - 支援者のための倫理的枠組み
3. **地域人材（Regional Human Resource）** - コミュニティベースの人材育成
4. **科学的アセスメント（Scientific Assessment）** - エビデンスに基づく評価手法

**関係性**: Three-Layer（哲学）→ Four-Pillar（実装）→ SoEシステム

## 主要概念

### 火文化圏思想

> 「記録されなかった問いは燃え尽きたものと見なされる。
> 記録されることでのみ、問いは再燃可能となる。」

- **記録主権**: 支援情報の所有権は被支援者に帰属する
- **技術実装**: YAML/Markdown形式での構造化記録

### 灰色の放棄倫理

福祉における二項対立的な倫理枠組みへの批判。支援関係の複雑な現実を認識する視点。

### 粒度型思考

粒度に基づく認知アプローチ：
- **Complex（複雑）** → **Complicated（込み入った）** → **Simple（単純）**
- システムレベル間のナビゲーションを可能にする
